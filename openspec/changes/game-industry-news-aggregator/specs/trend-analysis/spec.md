## ADDED Requirements

### Requirement: 关键词频率追踪
系统必须在可配置的时间窗口（每日、每周、每月）内追踪标签和分类的出现频率。频率数据必须被聚合并存储，用于历史趋势分析。

#### Scenario: 每日频率聚合
- **WHEN** 每日聚合任务在当天结束时运行
- **THEN** 系统必须计算并存储当天每个标签和每个分类的文章数量

#### Scenario: 历史频率查询
- **WHEN** 用户查询标签"Unreal Engine 5"在过去 30 天的出现频率
- **THEN** 系统必须返回该标签每日文章数量的时间序列

### Requirement: 上升趋势检测
系统必须通过比较当前时间窗口与上一时间窗口的关键词/标签频率来识别上升趋势。当某话题的频率增长超过配置的阈值（默认：较上一周期增长 50%）时，必须将其标记为"趋势话题"。

#### Scenario: 趋势话题识别
- **WHEN** 标签"AI NPC"上周出现在 5 篇文章中，本周出现在 15 篇文章中
- **THEN** 系统必须将"AI NPC"标记为趋势话题，并显示 200% 的增长指标

#### Scenario: 趋势话题列表
- **WHEN** 用户查看趋势页面
- **THEN** 系统必须展示按增长率排序的前 20 个趋势话题，显示当前数量、上期数量和百分比变化

#### Scenario: 自定义趋势阈值
- **WHEN** 管理员将趋势检测阈值设置为 100% 增长
- **THEN** 系统必须仅将增长超过 100% 的话题标记为趋势话题

### Requirement: 趋势可视化
系统必须为趋势数据提供可视化图表，包括单个话题的时间序列折线图和多个话题的对比图表。

#### Scenario: 单话题趋势图
- **WHEN** 用户选择特定标签或分类进行趋势分析
- **THEN** 系统必须展示折线图，显示所选时间范围（7 天/30 天/90 天）内的文章频率

#### Scenario: 多话题对比
- **WHEN** 用户选择 2-5 个标签进行对比
- **THEN** 系统必须展示叠加折线图，每个话题用不同颜色的线条表示，共享时间轴

### Requirement: 分类分布分析
系统必须提供指定时间段内文章在各分类中的分布细分，以比例图表形式展示。

#### Scenario: 分类分布视图
- **WHEN** 用户查看本周的分类分布
- **THEN** 系统必须展示饼图/环形图，显示每个分类的文章占比

#### Scenario: 分布随时间变化
- **WHEN** 用户比较两个时间段的分类分布
- **THEN** 系统必须展示并排的分布图表，突出显示发生显著变化的分类

### Requirement: 每周趋势摘要
系统必须使用 LLM API 生成每周趋势摘要，总结最显著的趋势、新出现的话题和趋于消退的话题。

#### Scenario: 自动摘要生成
- **WHEN** 每周摘要任务运行（可配置日期/时间，默认：周一 9:00）
- **THEN** 系统必须生成自然语言形式的本周关键趋势总结、每个分类的热门文章和显著变化，作为摘要文章存储，供所有团队成员查阅

#### Scenario: 摘要通知
- **WHEN** 新的每周摘要生成完成
- **THEN** 系统必须在仪表板上提供该摘要，并带有醒目的"新摘要"提示标识
