## ADDED Requirements

### Requirement: 预定义分类体系
系统必须维护一套预定义的分类体系用于文章归类。默认分类必须包括：AI 技术、游戏引擎、行业新闻、市场动态、开发工具、图形渲染、网络技术、公司动态。管理员必须能够添加、重命名和禁用分类。

#### Scenario: 默认分类可用
- **WHEN** 系统首次初始化
- **THEN** 系统必须创建默认分类体系，所有预定义分类均为激活状态

#### Scenario: 自定义分类添加
- **WHEN** 管理员向分类体系中添加新分类"程序化生成"
- **THEN** 系统必须注册该新分类，并在后续的 AI 分类提示词中包含该分类

#### Scenario: 分类禁用
- **WHEN** 管理员禁用某个分类
- **THEN** 系统必须停止将该分类分配给新文章，但保留其在已分类文章上的标记

### Requirement: AI 驱动的自动分类
系统必须使用配置的 LLM API 将每篇文章自动归类到分类体系中的一个或多个分类。分类必须在处理管线中的摘要生成之后执行。

#### Scenario: 单分类分配
- **WHEN** 一篇关于"Unity 新 AI 导航系统"的文章被处理
- **THEN** 系统必须基于内容分析分配"AI 技术"和"游戏引擎"等分类

#### Scenario: 多分类分配
- **WHEN** 一篇文章涉及跨多个分类的主题
- **THEN** 系统必须为每个分类分配置信度分数（0.0 至 1.0），仅存储高于配置阈值（默认：0.5）的分类

#### Scenario: 分类置信度阈值
- **WHEN** LLM 返回的分类分配置信度低于配置的阈值
- **THEN** 系统必须将该分类从文章的已分配分类中排除

### Requirement: 自动标签提取
系统必须使用 LLM API 为每篇文章提取和分配自由形式的标签。标签必须代表文章中提及的具体技术、公司、产品或概念（如 "Unreal Engine 5"、"NVIDIA DLSS"、"腾讯"、"强化学习"）。

#### Scenario: 标签提取
- **WHEN** 一篇关于 NVIDIA 游戏渲染新 GPU 的文章被处理
- **THEN** 系统必须提取"NVIDIA"、"GPU"、"游戏渲染"等标签以及提及的具体产品名称

#### Scenario: 标签归一化
- **WHEN** 从多篇文章中提取到指向同一实体但文本不同的标签（如 "UE5"、"Unreal Engine 5"、"虚幻引擎5"）
- **THEN** 系统必须将其归一化为规范标签形式，同时保留变体形式作为别名

### Requirement: 基于分类的筛选
系统必须允许用户按一个或多个分类筛选文章信息流，仅显示匹配所选分类的文章。

#### Scenario: 单分类筛选
- **WHEN** 用户选择"AI 技术"分类筛选器
- **THEN** 系统必须仅显示归类于"AI 技术"的文章，按时间倒序排列

#### Scenario: 多分类筛选
- **WHEN** 用户同时选择"AI 技术"和"游戏引擎"分类筛选器
- **THEN** 系统必须显示归类于任一分类的文章（OR 逻辑），按时间倒序排列

### Requirement: 基于标签的筛选
系统必须允许用户按标签筛选文章，支持包含和排除筛选。

#### Scenario: 标签包含筛选
- **WHEN** 用户点击标签"Unreal Engine 5"
- **THEN** 系统必须显示所有标记了"Unreal Engine 5"的文章

#### Scenario: 标签云展示
- **WHEN** 用户查看标签探索页面
- **THEN** 系统必须展示标签云，显示所选时间段内使用最频繁的标签，标签大小与频率成正比
