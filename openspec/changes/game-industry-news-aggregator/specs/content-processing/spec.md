## ADDED Requirements

### Requirement: 内容去重
系统必须使用双重机制检测和过滤跨所有数据源的重复文章：精确 URL 匹配和基于 SimHash 的内容相似度检测。

#### Scenario: 精确 URL 去重
- **WHEN** 抓取到的文章 URL 已存在于数据库中
- **THEN** 系统必须跳过该文章，并将其记录为基于 URL 的重复项

#### Scenario: 内容相似度去重
- **WHEN** 文章的 SimHash 指纹与现有文章的指纹汉明距离不超过 3 位
- **THEN** 系统必须将其标记为潜在重复项，关联到原始文章，并默认从主信息流中排除

#### Scenario: 去重阈值配置
- **WHEN** 管理员调整 SimHash 相似度阈值
- **THEN** 系统必须将新阈值应用于后续所有去重检查，不影响已处理的文章

### Requirement: 内容清洗
系统必须对所有数据源的原始内容进行清洗，移除 HTML 标签、多余空白、URL 中的追踪参数和广告内容，生成适合下游处理的干净纯文本。

#### Scenario: HTML 内容清洗
- **WHEN** 网页爬取的文章包含 HTML 标记
- **THEN** 系统必须剥离所有 HTML 标签，同时将有意义的文本结构（段落、列表）保留为纯文本

#### Scenario: URL 追踪参数剥离
- **WHEN** 文章 URL 包含追踪参数（utm_source、utm_medium、fbclid 等）
- **THEN** 系统必须移除这些参数，同时保留规范化的 URL

### Requirement: AI 驱动的自动摘要
系统必须使用 LLM API 为每篇文章生成简洁摘要（100-200 字）。摘要必须捕获与游戏行业专业人士和技术决策者相关的要点。

#### Scenario: 文章摘要生成
- **WHEN** 清洗后的文章进入处理管线
- **THEN** 系统必须将其发送至配置的 LLM API，并将返回的摘要（100-200 字）与文章元数据一并存储

#### Scenario: LLM API 失败回退
- **WHEN** LLM API 不可用或返回错误
- **THEN** 系统必须使用截断的摘录（干净内容的前 200 个字符）作为临时摘要存储文章，并将文章加入重试队列

#### Scenario: Token 预算强制执行
- **WHEN** 每日 LLM API token 使用量达到配置的预算上限
- **THEN** 系统必须停止发送新的摘要请求，对剩余文章使用截断摘录，并记录预算耗尽警告

### Requirement: 语言检测
系统必须自动检测每篇文章的语言（主要为中文和英文），并将检测到的语言作为元数据存储。

#### Scenario: 中文内容检测
- **WHEN** 文章主要包含中文文本
- **THEN** 系统必须为该文章标记语言代码 "zh"

#### Scenario: 英文内容检测
- **WHEN** 文章主要包含英文文本
- **THEN** 系统必须为该文章标记语言代码 "en"

### Requirement: 内容翻译
系统必须使用 LLM API 提供文章摘要在中英文之间的可选翻译。翻译必须根据用户偏好或系统配置触发。

#### Scenario: 英译中翻译
- **WHEN** 文章被检测为英文且系统配置了双语摘要
- **THEN** 系统必须生成摘要的中文翻译，并作为备选摘要存储

#### Scenario: 按需翻译
- **WHEN** 用户查看非偏好语言的文章摘要
- **THEN** 系统必须提供"翻译"操作，触发基于 LLM 的翻译并缓存结果

### Requirement: 处理管线编排
系统必须按顺序执行内容处理管线：清洗 → 去重 → 语言检测 → 摘要生成 → 翻译（如已配置）。每个阶段必须能够独立重试，无需重新处理前序阶段。

#### Scenario: 完整管线执行
- **WHEN** 一批原始文章进入处理管线
- **THEN** 系统必须按顺序逐阶段处理，每个阶段完成后存储中间状态

#### Scenario: 阶段级重试
- **WHEN** 某篇文章的摘要生成阶段失败，但清洗和去重已成功
- **THEN** 系统必须仅对该文章重试摘要生成阶段，无需重新执行清洗或去重

#### Scenario: 处理状态追踪
- **WHEN** 管理员查看处理仪表板
- **THEN** 系统必须显示当前管线状态，包括每个阶段待处理的文章数、处理速率和每阶段的错误数
